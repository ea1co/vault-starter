# vault-starter

A small, editable kit for setting up [Claude Cowork](https://www.claude.com/product/cowork) with a persistent, personal context file — what the kit calls a *vault*. The vault grows with you over time so every conversation with Claude builds on the last one instead of starting over.

This is v0.6. The kit is actively evolving. Things will change. Known-imperfect items live in `TODO.md`; the design reasoning lives in `LEARNINGS.md`.

## Getting started

1. Download or clone this repo.
2. Rename the folder to `vault-starter` and move it somewhere you'll remember (good default: `~/Documents/Projects/vault-starter`).
3. Open the folder and read `00_START_HERE.md` — it walks you through the Cowork setup (~5 minutes) and the first working session (~20 minutes).

That's the whole thing. Claude takes it from there.

## Who this is for

Anyone using Cowork who wants each conversation to build on the last — especially executives, consultants, and knowledge workers with ongoing bodies of work that benefit from Claude holding context across sessions.

## What's in the folder

- `00_START_HERE.md` — entry point. Start here.
- `ME.md` — the template Claude uses to create your personal vault.
- `01_WHERE_THIS_GOES.md` — direction of travel; where the kit is pointed longer-term.
- `02_WHEN_IT_GETS_WEIRD.md` — failure modes. Consult when something feels off.
- `03_WHY_THIS_IS_STRUCTURED_THIS_WAY.md` — optional theoretical frame.
- `INTEGRATION.md` — a sidequest methodology for reconciling multiple expert responses.
- `_example-mature-vault.md` — a composite-persona example of a populated vault.
